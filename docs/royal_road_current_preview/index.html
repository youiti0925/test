今回の改修ポイント (前回プレビューからの変更点)

  1. 各ケースの最上段に「状態 / 結論 / ENTRY / STOP / TP / RR / 次に待つ条件」だけを集約した compact 判断カード を追加。詳細パネルは下に <details> で折りたたみ。
  2. チャート上の文字は WNL / STOP / TP / EVENT のみ。 WB1 / WB2 / WBR / WPH / WFIB* 等の補助線は 線だけ描画して、ラベルは右側パネルに集約。
  3. 右側パネルに 5b. 未接続ファンダ を追加。 DXY / 米金利 (10y) / ニュース / 経済カレンダー を 赤(未接続) / 緑(接続済) で常時表示。
  4. 右側パネルに 6. 自動線一覧 (タップで非表示切替) を追加。スマホで 細い SVG 線を直接タップしなくても、ボタンで on/off できる。
  5. 右側パネルの 7. 手動線操作 に 「このブラウザ内 (localStorage) だけに保存。別端末との共有・正式保存には annotations.json のダウンロードが必要」の警告を高コントラストで表示。
  6. pattern_levels の STOP / TP が W ライン由来でなく skeleton から補完された 場合、判断カード直下に橙色の警告 (level_source = skeleton_fallback) を表示。
  7. 波形 → pattern_levels → entry_plan → integrated action の 間接決定経路 を wave_derived_lines.py docstring と 画面表示で明示。

このプレビューでユーザーに確認してほしいこと

  1. 各ケースの一番上に 状態 / 結論 / ENTRY / STOP / TP / RR / 次に待つ条件 が まとまって読めるか (チャートを見る前にそれだけで判断できるか)
  2. チャート上に余計な記号 (WB1/WB2/WBR/WPH/WFIB*) が ないか — あるのは WNL/STOP/TP/EVENT だけのはず
  3. 右側パネルの 5b. 未接続ファンダ が DXY/米金利/ニュース/経済カレンダー の状態を 赤/緑で見せているか
  4. 右側パネルの 6. 自動線一覧 から [非表示] ボタンでチャート上の線を 消せるか (スマホでもタップしやすいか)
  5. 右側パネルの 7. 手動線操作 に「ブラウザ内のみ保存」の警告が ハッキリ見えるか
  6. WAIT_RETEST のケースで 「まだ入らない理由」 (戻り (リターンムーブ) 待ち / 確定足待ち) が読めるか
  7. WAIT_EVENT_CLEAR のケースで 赤い縦バンド + 右パネルに FOMC + BLOCK 表示が出ているか
  8. STOP / TP が skeleton から補完されたケースで 橙色の補完警告 が 判断カード直下に出ているか

凡例 (判断カードの状態)

状態結論行意味
READY 入れる (READY) 波形 + 直近サポレジ + 確定足 + 最小 RR (≥ 2) を満たしている。
WAIT_BREAKOUT まだ待つ — ブレイク待ち 波形は認識されたが、ネックライン (WNL) をまだ越えていない。
WAIT_RETEST まだ待つ — 戻り (リターンムーブ) 待ち WNL を一度抜けたが、戻りと確定足がまだ来ていない。
WAIT_EVENT_CLEAR イベント待ち (ファンダ・リスク) READY だがイベント (FOMC など) が窓内にあるためダウングレード。
HOLD 見送り (HOLD) 条件不足・root 拒否などで条件を満たしていない。

波形 → 判断 までの間接経路 (重要)

画面上で used_in_decision = false と書かれている W ライン (raw な波形由来の自動線) そのものは、最終 BUY/SELL/HOLD を直接変更しません。 ただし integrated profile では: 

raw W ライン  →  pattern_levels  →  entry_plan  →  integrated action

の経路で 間接的に判断に使われています。 判断カードの ENTRY / STOP / TP / RR は pattern_levels 経由の値で、 その元データは W ラインまたは skeleton 補完です (skeleton 補完だった場合は 判断カード直下に橙色警告が出ます)。

まだ未接続 / 未完成な項目

  • Daily 10k preset: まだ着手していない (本プレビューの後段で別タスク)
  • DXY / 米金利 / ニュース / 経済カレンダー の実フィード: 未接続 (合成データのため 5b パネルでは「未接続」表示)
  • イベントカレンダーは 合成 FOMC イベント 1 件 を注入している (WAIT_EVENT_CLEAR デモ用) — 実カレンダーへの接続は別途
  • 手動線の永続化 は localStorage のみ (サーバ送信なし、デバイス間共有なし、annotations.json で持ち出し)

本プレビューは観測専用の opt-in プロファイル royal_road_decision_v2_integrated の出力です。 既存の本番系 (live / OANDA / paper / current_runtime CLI 既定) は 一切変更されていません。
