このプレビューでユーザーに確認してほしいこと

  1. ダブルボトム / ダブルトップで ENTRY / STOP / TP と WNL / WSL / WTP がチャートに正しく描かれているか
  2. 右サイドの 6 行情報パネル (現在の状態 / エントリープラン / ファンダ・イベント / 王道根拠 / 未接続 / 手動線操作) が読めるか
  3. entry_plan が状況に応じて READY WAIT_BREAKOUT WAIT_RETEST WAIT_EVENT_CLEAR HOLD と切り替わっているか
  4. イベントリスク (FOMC など) が来ているとき、 チャート上に 赤い縦バンド が描かれ、最終アクションが HOLD (WAIT_EVENT_CLEAR) にダウングレードされているか
  5. 手動線 UI (追加 / エクスポート) が見えているか — 既定は display_only で、ユーザーが描いた線は 判断には使われないこと
  6. 本番系 (live / OANDA / paper) は触っていないこと (このプレビューは観測専用 opt-in プロファイル)

凡例

状態意味
READY 波形 + 直近サポレジ + 確定足が揃っており、 最小 RR (≥ 2) も満たしている。エントリーが推奨される状態。
WAIT_BREAKOUT 波形は認識されたが、ネックライン (WNL) をまだ越えていない。 待機 (HOLD)。
WAIT_RETEST WNL を一度抜けたが、戻りと確定足がまだ来ていない。待機 (HOLD)。
WAIT_EVENT_CLEAR READY だったがイベント (FOMC など) が窓内にあるため ダウングレード。待機 (HOLD)。
HOLD 波形未確認・RR 不足・root 拒否などで条件を満たしていない。

まだ未接続 / 未完成な項目

  • Daily 10k preset: まだ着手していない (本プレビューの後段で別タスク)
  • 合成 OHLC のため マクロ系 (macro_alignment / vix etc.) は限定的な値で表示される
  • イベントカレンダーは合成イベント (FOMC) を 1 件注入している — 実カレンダーへの接続は別途
  • 手動線の永続化 は localStorage のみ (サーバ送信なし、デバイス間共有なし)
