★ 最新追加: 「この判断の読み方 (decision bridge)」

各ケースの最上段に 「★ この判断の読み方」 が追加されました。
表示されている情報を 4 段階に分類し、最終 BUY / SELL / HOLD と どう結びついているかが一目で分かります:
USED (緑) = 判断に使用 — royal_road_v2 本体, block_reasons, stop_plan
PARTIAL (青) = 一部使用 — support_resistance_v2 など
AUDIT_ONLY (黄) = 表示のみ — 波形認識, Wライン, フィボ, Masterclass 19 パネル, Daily Roadmap
NOT_CONNECTED (赤) = 未接続 — ファンダ実データ, position sizing, 経済指標, 通貨ペア固有ファンダ, 下位足解剖
HOLD ケースには 「入れそうに見えるかもしれません」 + 理由表示。
BUY/SELL ケースには 「波形・フィボ・Masterclass は直接使っていません」 注意表示。

Source-pack 19 パネル + 3段表示 (observation-only)

各ケースに Masterclass + Source-pack 19 機能パネル が 3 段構成 で表示されます:
① 重要サマリ (今日このケースで見るべきこと)
② 王道チェック (13 軸 グランドコンフルエンス: Dow / Line / MA / Oscillator / Price Action / Pattern / RR / Invalidation / Macro / Fibonacci / MTF / Roadmap / Symbol-macro)
③ 詳細パネル (16 Masterclass + 3 Source-pack: Fibonacci_Trading_Blueprint / Daily_10k_FX_Roadmap / USDJPY_Tactical_Briefing)
チャート上には WFIB382 / WFIB500 / WFIB618 / WFIB1272 / WFIB1618 のフィボナッチ線が描かれます。
すべて observation-only / used_in_decision = false で、 売買判断 (BUY / SELL / HOLD) には接続されていません。
ポジションサイジング / 経済指標カレンダー / 通貨ペア固有ファンダ実データは 未接続 (UNKNOWN) として明示されます。

この画面で確認すること

この画面は 売買成績 を見るものではありません。
王道判断をシステムがどう再現しているか を、 人間の目で確認するための画面です。

確認すること:

  1. サポート / レジスタンス帯 (緑/赤の帯) が、自然な位置にあるか
  2. トレンドライン (青い線) が無理やり引かれていないか
  3. 損切り線 (stop / structure_stop) が「ここを割ったらシナリオ崩壊」と思える位置か
  4. 王道チェックの OK (PASS) / 注意 (WARN) / 見送り (BLOCK) の理由が納得できるか
  5. 現行ロジック (current_runtime) と王道v2 (royal_road_v2) の差分が分かるか

おかしい線・無理やりな根拠・納得できない判断があれば「修正対象」です。 気づいた点は feedback_template.json に記入する想定です。

パターン部位チャートデモ (4 種)

波形認識からの線 (WNL/WB1/WB2/WP1/WP2/WLS/WH/WRS/WSL/WTP) が チャート上に出るかを 4 つの典型パターンで確認できます。 どれも合成 OHLC を使った決定論的デモ (バックテスト結果ではありません)。

① ダブルボトムデモ (double_bottom) 合成パターン

見る目的: DB1 / B1 / B2 / NL / BR / WSL / WTP の線・ラベルが出るか。

  • チャート上に紫の WNL1 ネックライン が水平に引かれている
  • 赤の WSL1 (波形損切り候補) と緑の WTP1 (利確候補) が出る
  • 「波形だけチャート」に DB1 家族マーカー + B1/B2/NL ラベル
  • タイトルが「判断: HOLD / 波形候補: ダブルボトム候補 / 参考線: ...」
② ダブルトップデモ (double_top) 合成パターン

見る目的: DT1 / P1 / P2 / NL / BR / WSL / WTP の線・ラベルが出るか。

  • P1 / P2 の白枠ラベル + WNL1 ネックライン (紫横線) が出る
  • WSL1 = P2 の上、WTP1 = NL から下方向に投影
  • 波形候補欄に ダブルトップ候補
③ 三尊デモ (head_and_shoulders) 合成パターン

見る目的: HS1 / LS / H / RS / NL / BR / WSL / WTP の線・ラベルが出るか。

  • 左肩 (LS) / 頭 (H) / 右肩 (RS) のラベルが pivot 位置に出る
  • WNL1 ネックライン (左右の谷を結んだ位置)
  • WSL1 は右肩の上、WTP1 は NL - (頭 - NL)
④ 逆三尊デモ (inverse_head_and_shoulders) 合成パターン

見る目的: IHS1 / LS / H / RS / NL / BR / WSL / WTP の線・ラベルが出るか。

  • 左肩 (LS) / 頭 (H、最も深い底) / 右肩 (RS) ラベル
  • WNL1 ネックライン (左右のピークを結んだ位置)
  • WSL1 は右肩の下、WTP1 は NL + (NL - 頭)

その他のサンプル (3 種)

⑤ 通常 v2 レポートサンプル (normal v2 report)

見る目的: ランダムウォーク合成データで、波形認識が triangle/wedge/flag を拾えるか。

  • サポレジ帯 (緑/赤) + 波形由来ライン (WUP/WLOW/WBR/WSL/WTP) が混在
  • 線カウントサマリ表で「採用 / 不採用 / 波形由来」 の合計が見える
  • HOLD ケースでは「未ブレイク」を示す注意表示

※ 合成 OHLC を用いたサンプル (live データではありません)。

⑥ 損切りデモ (structure stop anchor demo) 手作りフィクスチャ

見る目的: サポート帯と損切り線が合っているかを見る (波形認識を使わない既存系のデモ)。

  • 緑の帯がサポート帯
  • stop / structure_stop の線が、その帯の下または近くにあるか
  • 損切り根拠の説明に sr_zone:support と書かれているか
⑦ 現行 vs 王道 差分デモ (current vs royal diff demo) 手作りフィクスチャ

見る目的: 現行ロジックがBUYした場面を、王道v2がHOLDで止めた理由を見る。

  • 比較表で 現行ロジック = BUY / 王道v2 = HOLD
  • 差分タイプが current_buy_royal_hold
  • ブロック理由 (block_reasons) が並んでいて、王道v2が止めた理由が納得できるか
